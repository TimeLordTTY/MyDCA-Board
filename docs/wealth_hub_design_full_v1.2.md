# 财富中枢系统 · 总体功能设计文档v1.2

---

## 1. 系统总体概述

### 1.1 系统定位

“财富中枢”是一个供个人长期使用的 **统一资产管理、定投策略、数据分析与 AI 投顾系统**。
目标不是简单记账，而是构建一个可增长、可扩展、可自动化的 **个人金融操作系统（Personal Financial OS）**。

### 1.2 产品目标

* 全面管理主人的所有金融账户与资产
* 自动化定投、监控、回撤修复、加仓提示
* 构建数据驱动的决策系统
* 让主人能“一眼看到全部财富状态”
* 提供基于主人真实数据的 AI 咨询能力
* 最终成为一个可陪伴 10 年以上的财富基础设施
* 可追溯、可重算、可对账、可解释

### 1.3 用户画像

* 主人本人，主要决策者
* 具备一定理财知识（正在提升中）
* 具备中长期投资目标
* 对策略模拟、趋势判断、科学理财有兴趣
* 熟悉 Python / AI / 定投策略迭代

# 3. 各模块详细功能说明（可直接用于技术方案）

## 3.1 资产账户管理系统（核心骨架）

### 功能目标

统一管理主人所有金融资产，形成“个人版资产负债表”。

### 账户类型

* **现金类**：银行卡、余额宝、活期
* **固收类**：短债、银行理财、北银系列
* **权益类**：基金（LOF/ETF/场外）、股票
* **特殊资产**：黄金、外汇等

（补充：资产负债表不仅要能看“资产”，还要能看“净资产”。因此负债模块在 v1.2 明确纳入“资产账户管理”的范畴，见下方“资产负债与流动性视角”。）

### 主要功能

* 手工录入 + 自动同步
* 每日自动刷新净值
* 计算收益率
* 计算年度年化收益率
* 计算最大回撤与修复期
* 自动分类汇总（固收/权益/黄金）
* 账户权重分析
* 数据来源标记（来源域 / 采集方式 / 可信等级）
* 关键字段证据链（保留原始响应、抓取时间、解析版本）
* 异常数据隔离（不污染主表：进入 quarantine/异常区，等待人工确认）
* 人工修正入口（对账修正必须落入“修正账本/修正日志”，避免暗改数据）

### 数据口径与时间语义（关键约束）

为避免 QDII/海外资产/银行理财等“净值滞后”导致账面错觉，系统统一定义：

* **采集日（Fetch Date）**：系统运行当天生成快照的日期（账户视角的“今天”）
* **净值日（NAV Date）**：产品官方披露净值对应的日期（产品视角的“估值日”）
* **交易日（Trade Date）**：买入/卖出/申赎/到账的日期（现金流视角的“行为日”）

在展示层：

* “今日资产/今日盈亏”默认按 **采集日** 汇总
* “净值曲线/产品回撤/对账”按 **净值日** 展示
* 采集时间（fetch_at）
* 数据有效时间（nav_date）
* “数据是否为滞后净值”的提示标签（例如：QDII T+1、理财非交易日等）

### 收益率与绩效指标口径（必须统一）

系统内涉及“收益率/年化/绩效”时，需明确采用指标：

* **简单收益率**：`(现值 - 成本) / 成本`（仅做静态快照参考，不适合评价定投绩效）
* **资金加权收益率（MWR/XIRR）**：个人最重要指标（考虑现金流时间价值，定投评价以此为准）
* **时间加权收益率（TWR）**：用于策略对比/回测（剔除资金进出影响，衡量策略本身）
* Dashboard 默认展示 **XIRR（MWR）** 作为“真实收益率”，简单收益率仅做参考
* XIRR 计算必须基于 **交易账本现金流**，且要明确现金流符号约定（投入为负、回收为正，或反之，但全局一致）
* 对于无法补齐历史流水的资产，允许设置“期初估值/期初成本”作为 XIRR 的起点，但必须标注“口径：估算”

### 交易账本（Transaction Ledger）作为“唯一真相源”

资产账户管理必须以“交易事实”驱动，而非仅靠静态持仓：

* 系统以 **交易账本** 作为唯一真相源（买入/卖出/分红/手续费/转换）
* **持仓份额（units）与成本（cost）由账本推导**
* 静态持仓仅作为过渡或兜底（例如：历史遗留、无法补齐流水的资产）

交易类型需至少覆盖：

* BUY / SELL
* DIVIDEND_CASH（现金分红）
* DIVIDEND_REINVEST（红利再投）
* FEE（手续费）
* TRANSFER_IN / TRANSFER_OUT（账户间资金转移）
* action_time（可选，精确到分钟，用于同日多笔排序）
* settlement_date（可选，到账/确认份额日期，用于“份额下午才到账”的场景）
* channel / broker（支付宝/银行/券商/网商等）
* external_ref（外部流水号/订单号，可为空，但保留字段）
* source（manual/import/api），用于区分人工补录与自动导入

### 分红与公司行为处理（必须支持）

长期收益口径必须正确处理分红：

* **现金分红**：不改变份额，形成正现金流（计入 XIRR）
* **红利再投**：份额增加，净值除权，不视为新增投入

（如不处理，将造成“净值突然下跌但实际未亏”的误判。）
* SPLIT / MERGE（拆分 / 合并）
* SWITCH（基金转换）
* REDEEM_FEE / SUBSCRIBE_FEE（申购赎回费拆分）

## 3.2 自动定投引擎（SIP Engine）

### 功能目标

完全复刻主人当前的真实定投体系，同时支持未来策略扩展。

### 功能点

* 每日/每周/每月定投计划
* 自动判断本周期应该投多少钱
* 自动检测是否进入回撤阶段
* 自动加仓 / 暂停
* 支持接入策略（如 MA、回撤、盈利池等）
* 显示未来 12 月投多少钱的计划图
* 今日建议清单（产品、建议金额、触发理由、参考指标、置信等级）
* 策略执行日志（每一次建议都必须记录：输入数据、参数、输出结果）
* “不建议交易”也要有输出（避免主人看到空白误以为系统挂了）

### 数据处理

* 定投金额
* 现金池余额
* 策略触发日志（非常重要）

### 与账本的关系（口径约束）

* 定投引擎输出的是“建议/信号/计划”，真实执行后必须落入交易账本
* 定投执行与复盘、收益率计算，应以账本现金流为准（支持 XIRR）
* 建议 → 执行确认（主人点击确认/导入流水） → 自动落账（写入交易账本） → 更新持仓与成本

---

## 3.3 行情抓取与数据同步（Data Sync）

### 数据源

* 天天基金 API
* ETF 场内交易价格
* 黄金实时价格
* 纳指、沪深300、大盘指数
* 银行固收收益（爬虫或 API）

### 支持功能

* 自动每日拉取
* 自动缓存
* 缺失数据补全
* 异常监控
* 多数据源兜底（同一资产可配置主源/备源，主源失败自动切换）
* 解析器版本化（HTML/接口变更时可快速定位是哪一版解析失效）
* 限流与重试策略（避免被封/避免瞬时失败导致全盘缺数据）
* 采集成功率监控与告警（按产品、按来源统计）

### 数据质量与鲁棒性要求

* 数据源可能滞后：系统需支持“回溯补齐”与“缺失重试”
* 数据源可能变更：采集层需可替换（适配器/客户端模式）
* 关键字段必须校验（产品代码、日期、净值、累计净值、增长率等）
* 日期必须可解析且在合理范围（不得大于采集日 + 1）
* 净值必须为正数且满足合理区间（可配置）
* 同一产品同一净值日若出现多次不同净值：必须进入异常区，禁止直接覆盖（需人工确认或依据“官方优先级”规则处理）

## 3.4 财富分析中心（分析图表）

### 数据展示

* 资产总额趋势图
* 各类资产占比（饼图）
* 单基金收益曲线
* 定投成本 vs 市值曲线
* 最大回撤与修复可视化
* 资产波动率计算

### 典型分析视图

* “我一年赚了多少？”
* “我总共投了多少钱？”
* “收益靠哪只基金贡献？”
* “现在的风险高吗？”

### 分析指标口径补充（必须明确）

为保证分析可信，建议将以下指标纳入标准输出：

* 累计投入（本金）
* 当前市值
* 未实现盈亏（浮动盈亏）
* 已实现盈亏（卖出/现金分红兑现部分）
* 账户视角“当日盈亏”（按采集日，参考上一次采集快照）
* XIRR（默认绩效指标）
* TWR（策略对比/回测指标）
* 基准（Benchmark）：沪深300 / 中证全债 / 纳指等（可配置）
* 超额收益（Alpha）：投资组合收益 - 基准收益
* 波动对比（Beta/相关性）：用于判断是否“看似分散，实则同涨同跌”

# 6. 现金流预算系统（与现实账户同步）

### 功能用途

让主人知道“本月真实可投资金额是多少”。

### 功能点

* 收入记录
* 固定支出（房租、餐饮等）
* 投资预算自动生成
* 消费 vs 投资偏差警告

### 边界定义（避免系统被“生活记账”拖累）

* 财富中枢重点管理“投资性现金流”（从工资/活钱流入投资账户的那一刻开始）
* 生活支出可采用“月度汇总”而非逐笔记账（降低维护成本，提高长期可持续性）
* 安全金下限（可配置）：低于阈值时，定投建议自动降档/暂停，并给出原因
* 近期刚性支出覆盖率：未来 N 天刚性支出是否可被现金+短期固收覆盖

# 7. 策略辅助系统（核心特色之一）

## 7.1 策略模拟（What-if Analysis）

* “假如景顺下个月涨 10%，我现在买多少最好？”
* “如果固收占比提高到 40% 会怎样？”
* 自定义仓位模拟
* 基于历史数据的策略重跑

### 建议的落地方式

* 以“历史重演/回测”为主（可验证）
* 情景模拟以“参数化敏感性分析”为主，不输出确定性预测承诺
* 明确情景模拟的输出必须包含“假设条件”，并且所有假设可编辑、可复现

## 7.2 回测系统

* 导入历史净值
* 套用策略（MA、回撤加仓、盈利池等）
* 输出：收益率、最大回撤、修复期
* 可对比 Pure SIP 与策略表现

### 回测指标口径

* TWR：用于策略本身表现对比
* XIRR：用于模拟“主人真实投入节奏”的个人收益体验

# 8. 风险监控系统

## 风险来源

* 市场高估（指数估值）
* 大盘回撤
* 单基金波动异常
* 固收断崖
* 现金比例过低

## 风险提示

* 超过某阈值自动弹提醒：“主人，风险区间来了喔，要注意心态～”
* 定投暂停机制

### 资产负债与流动性视角（增强项）

* 长期可增加负债管理（房贷/分期等），用于计算净资产与资产负债率
* 重点监控流动性：现金与短期固收是否覆盖近期刚性支出
* 负债台账（本金余额、利率、还款日、月供、剩余期数）
* 资产负债率（负债 / 资产）
* 净资产趋势（资产 - 负债）

# 9. AI 模块（高级扩展）

## 9.1 AI 趋势分析

* 基于主人真实数据进行**变化解释与归因**（为什么今天涨/跌）
* 识别净值滞后带来的“错觉变化”，生成可读解释
* 结合风险规则输出“风险/震荡/平稳”状态标签
* 输出建议必须可追溯：引用到对应持仓与指标
* AI 的每条关键结论需能映射到数据点或规则触发（否则标注为“推测/建议”，且降低权重）
（AI 不做承诺性预测，不替代主人决策；更像“解释器 + 复盘助手”。）

## 9.2 自然语言问答

示例：

* “帮我看一下景顺今天值不值得买？”
* “我本周收益情况怎么样？”
* “我距离 100 万还差多少？”

系统基于主人个人真实数据回答，而非大众信息。

## 9.3 财富规划助手

* 买房规划
* 婚礼预算
* 养老金测算
* 长期资产增长预测

（规划类输出应基于可解释假设与情景参数，不做确定性保证。）
