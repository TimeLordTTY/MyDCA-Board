# 统一记账入口 - 全链路修改总结

## 修改概述

本次修改围绕3个核心问题，对DDL、设计文档、开发实施指南、PC Demo进行了全链路一致更新：

1. **问题1**：记账功能"一个入口"覆盖所有业务类型
2. **问题2**：国债逆回购归类与记账方式（现金增强方案）
3. **问题3**：专项资金（RESERVED）允许做逆回购"买入"

## 已完成的修改

### 1. DDL修改（`sql/initsql/01_DDL_v2_revised.sql`）

#### 1.1 扩展`txn_type`枚举
- 新增：`CUSTODY_TRANSFER`（转托管）
- 新增：`BOND_REPO`（国债逆回购）

#### 1.2 扩展`relation_type`枚举
- 新增：`REFUND_OF`（退款属于原交易，推荐）
- 新增：`REIMBURSEMENT_OF`（报销属于原支出，推荐）
- 新增：`CUSTODY_TRANSFER_OF`（转托管属于原事件）
- 保留：`REFUND`、`REIMBURSE`（兼容旧值）

#### 1.3 补丁文件说明
- `02_DDL_patch_transaction_linking_and_split_payment.sql`：已标记为"已合并到主DDL"，仅作历史参考

### 2. 设计文档修改（`docs/财富中枢系统完整设计方案.md`）

#### 2.1 新增"统一记账入口"章节（4.3.3节）
- **支持的业务类型**：
  - 日常类：EXPENSE、INCOME、TRANSFER
  - 资金类：REFUND、REIMBURSE（必须关联原交易）
  - 理财交易类：BUY、SELL、SUBSCRIPTION、REDEMPTION、CUSTODY_TRANSFER、BOND_REPO

- **UI交互流程**：
  - 第一步：选择业务类型
  - 第二步：根据类型展示不同字段
  - 第三步：生成统一数据结构

- **关键业务规则**：
  - 退款/报销必须关联原交易
  - 组合支付支持多个账户共同支付
  - 转托管无现金流，只迁移持仓和成本
  - 逆回购不生成持仓，只占用资金和生成利息

#### 2.2 更新"交易关联关系模型"章节（4.3.4节）
- 扩展`relation_type`枚举说明
- 新增转托管关联场景（CUSTODY_TRANSFER_OF）
- 明确关联优先规则：`related_txn_id`用于语义关联，`biz_group_key`用于UI聚合

#### 2.3 新增"逆回购现金增强方案"说明（4.4.2节）
- **归类决策**：采用B方案（现金增强工具），不生成持仓POSITION
- **资金来源规则**：
  - 允许从任意`fund_usage`账户（SPENDABLE/RESERVED/INVESTABLE）作为资金来源
  - **为什么允许RESERVED账户做逆回购**：逆回购为短期现金增强（1天期），且会在真正支出前提前一天出金到余利宝，不影响专款用途
  - 但仍要严格禁止：RESERVED账户用于日常EXPENSE消费，RESERVED账户用于ETF/股票等风险投资

- **下单处理**：创建REPO订单（PENDING），仅增加`reserved_amount`，不生成流水
- **到期处理**：释放`reserved_amount`，生成利息流水（CASH DEBIT + INCOME CREDIT）
- **看板展示口径**：`repo_locked_amount`（从订单汇总）、`repo_interest_sum`（从利息流水汇总）
- **端到端示例**：完整的从创建到到期确认的示例

### 3. 开发实施指南修改（`docs/开发实施指南.md`）

#### 3.1 更新Phase 1.3：统一记账模块
- **LedgerTxn实体字段**：更新`txnType`和`relationType`枚举值
- **LedgerService核心方法**：
  - `createRefund`：使用`REFUND_OF`（推荐）或`REFUND`（兼容）
  - `createReimburse`：使用`REIMBURSEMENT_OF`（推荐）或`REIMBURSE`（兼容）
- **QuickEntryService方法**：
  - 新增`createUnifiedTransaction`（统一记账入口，覆盖所有业务类型）
  - 支持组合支付：`paymentLines`参数
- **LedgerController接口**：
  - `POST /api/v2/ledger/txns`：统一记账入口，支持所有业务类型

#### 3.2 更新Phase 1.4：订单与结算模块
- **OrderService核心方法**：
  - `createOrder`：更新`fund_usage`校验规则
    - EXPENSE订单：必须=SPENDABLE
    - BUY/SUBSCRIPTION订单（非逆回购）：必须=INVESTABLE
    - BOND_REPO订单：忽略fund_usage限制，允许SPENDABLE/RESERVED/INVESTABLE
  - 新增逆回购特殊处理说明
- **SettlementService核心方法**：
  - `confirmSettlement`：区分普通投资订单和逆回购订单的处理逻辑
  - 逆回购订单：释放`reserved_amount`，生成利息流水，不生成POSITION分录

### 4. PC Demo修改（`web/demo-pc-dashboard.html`）

#### 4.1 Sticky顶栏
- Header区域改为sticky定位，滚动时固定
- 标题说明文字放到sticky下方

#### 4.2 统一记账入口
- 将"快速支出"和"快速收入"按钮合并为一个"记一笔"按钮
- 创建统一记账中心弹窗/抽屉
- **第一步：选择业务类型**
  - 11种业务类型按钮：支出、收入、转账、退款、报销、买入、卖出、申购、赎回、转托管、逆回购
- **第二步：根据类型展示不同字段**
  - 支出/收入：账户、金额、备注、组合支付（可选）
  - 退款/报销：原交易选择、退款/报销金额、到账账户、显示已退款/已报销金额
  - 逆回购：产品选择、账户选择（允许RESERVED）、本金、到期日期
  - 其他投资类：产品、账户、金额/份额
- **第三步：生成统一数据结构**
  - 本地假数据模拟提交
  - 更新交易流水列表

#### 4.3 交互功能
- 退款/报销：显示原交易金额、已退款/已报销金额、剩余可退/可报金额
- 组合支付：支持添加多个资金来源行
- 逆回购：明确提示允许RESERVED账户，并说明原因
